Voz & Tono

Cómo habla Guai

La comunicación de Guai es directa, experta y humana. Hablamos de resultados, no de tecnología por la tecnología.

Directo y concreto

Evitamos jerga innecesaria. Decimos "reduce 75% del tiempo" no "optimiza los flujos de trabajo mediante automatización avanzada". Los números concretos ganan siempre.

Cercano sin ser informal

Tuteamos al usuario. Somos accesibles y empáticos, pero mantenemos la precisión técnica. No usamos emojis en interfaces de producto excepto en contextos muy específicos.

Enfocado en el valor

Siempre comunicamos el "¿qué ganas tú?" antes de explicar el "¿cómo funciona?". El usuario primero quiere saber el beneficio.

Confiado, nunca arrogante

Afirmamos capacidades con evidencia. Evitamos hipérboles vacías como "la mejor solución del mercado". Preferimos "clientes que reducen costos 3x en 30 días".

✓ Sí decimos
"Automatiza en 48 horas, sin código"
"Tu equipo se enfoca en lo que importa"
"Reducimos un 65% el tiempo de onboarding"
✗ No decimos
"Solución de IA de clase mundial disruptiva"
"Potenciamos sinergias con machine learning"
"¡¡La mejor plataforma del mercado!!"

Principio: Nunca usar animaciones por decoración pura. Cada movimiento debe comunicar estado, dirección o retroalimentación. Reducir motion si el usuario lo prefiere con prefers-reduced-motion.

14
Guidelines

Do's & Don'ts

Reglas visuales clave para mantener coherencia en cualquier pieza de comunicación.

✓ Hacer
✓ Usar siempre fondos oscuros (#0A0C0F o similares) como base
✓ Reservar el verde (#00E5A0) para elementos interactivos y CTAs
✓ Usar Syne para todos los títulos y DM Sans para cuerpo
✓ Mantener padding mínimo de 28px en todos los contenedores
✓ Incluir siempre la capa de noise texture en páginas principales
✓ Usar un solo botón Primary por pantalla o sección
✗ Evitar
✗ Usar gradientes de color brillante sobre fondos claros (antiestético a la marca)
✗ Saturar interfaces con el color verde (perderá su jerarquía)
✗ Usar fuentes distintas a Syne y DM Sans sin aprobación
✗ Colocar texto blanco puro (#FFFFFF) — usar siempre #F0F2F5
✗ Crear botones primarios con esquinas completamente rectas
✗ Animar elementos sin propósito funcional claro

Design Tokens

CSS Variables

El código fuente de verdad. Copiar este bloque en el :root de cualquier proyecto Guai para heredar toda la identidad.

/* ── GUAI DESIGN TOKENS v1.0 ── */ :root { /* Backgrounds */ --bg: #0A0C0F; --bg-card: #111318; --bg-raised: #16191F; /* Borders */ --border: rgba(255,255,255,0.07); --border-med: rgba(255,255,255,0.12); /* Brand Colors */ --accent: #00E5A0; --accent-dim: rgba(0,229,160,0.12); --accent-glow:rgba(0,229,160,0.25); --yellow: #F5C842; --yellow-dim: rgba(245,200,66,0.12); --blue: #5B8DEF; --blue-dim: rgba(91,141,239,0.12); /* Text Scale */ --text-1: #F0F2F5; /* Primary */ --text-2: #8D939F; /* Secondary */ --text-3: #555C68; /* Tertiary / Disabled */ /* Shape */ --radius: 14px; --radius-lg: 24px; --radius-pill:100px; /* Typography */ --font-head: 'Syne', sans-serif; --font-body: 'DM Sans', sans-serif; /* Shadows */ --shadow-card: 0 2px 20px rgba(0,0,0,0.4); --shadow-glow: 0 0 40px rgba(0,229,160,0.15); /* Transitions */ --t-fast: 0.15s ease; --t-base: 0.2s ease; --t-slow: 0.3s ease; }
